# Security Fixes for SES Think Tank

Fix the following security issues. Do NOT modify test files. After each fix, verify the file still imports correctly. DO NOT run tests.

## 1. tools/__init__.py - Sandbox bypass (CRITICAL)

The sandbox at lines 261-332 is a regex blacklist that's trivially bypassed. Replace with a whitelist approach. Block: `__import__`, `__builtins__`, `exec`, `eval`, `compile`, `open`, `input`, `getattr`, `setattr`, `delattr`, `vars`, `dir`, `globals`, `locals`, `__subclasses__`, `__mro__`, `__base__`, `ctypes`, `os.system`, `subprocess`, `socket`, `shutil`, `sys.modules`, `sys.exit`.

## 2. plugins/tools/__init__.py - Arbitrary code execution (CRITICAL)

`_exec_python` at lines 65-78 runs string-substituted code with full builtins. Fix by:
- Using `json.dumps()` for argument substitution instead of string replacement
- Running in a subprocess with restricted globals (no `__builtins__` or a minimal dict with only safe builtins like `print`, `len`, `str`, `int`, `float`, `list`, `dict`, `tuple`, `set`, `bool`, `range`, `enumerate`, `zip`, `map`, `filter`, `sorted`, `reversed`, `any`, `all`, `sum`, `min`, `max`, `abs`, `round`)

## 3. auth.py - JWT secret not persisted (HIGH)

Line 25: JWT secret regenerated on every restart if `SES_JWT_SECRET` unset. Persist to disk at `data/.jwt_secret`. Read on startup, write if not exists.

## 4. auth.py - Hardcoded default credentials (HIGH)

Line 491: Hardcoded `admin`/`admin123` default credentials. Auto-generate a random password using `secrets.token_urlsafe(16)` and print it on first run.

## 5. settings.py - Encryption key not persisted (HIGH)

Line 23: Encryption key regenerated per restart. Persist like JWT secret at `data/.encryption_key`.

## 6. routers/platform.py - No auth on session routes (HIGH)

Lines 211-474: No auth on session routes. Anyone can read/delete/intervene on any session by guessing the 8-char ID. Add `get_current_user` dependency to the session detail routes (get_session, update_session, delete_session, intervene).

## 7. routers/platform.py - Non-admin can approve plugins (HIGH)

Line 109: `approve_plugin_api` only checks `get_current_user`, not `require_admin`. Swap to `require_admin`.

## 8. export.py - Private shares are public (HIGH)

Lines 152-189: `is_public` flag never checked before returning content. Add `if not share["is_public"]: raise HTTPException(403)`.

## 9. tools/__init__.py - Path traversal (MEDIUM)

Line 350: `startswith()` without separator is bypassable. Use `resolved.is_relative_to(project_root)` instead.

## 10. auth.py - Weak password hash (MEDIUM)

Lines 96-107: Single HMAC-SHA256, no KDF. Use `hashlib.pbkdf2_hmac('sha256', password.encode(), salt, 100000)` instead.
